# 빠른 시작

설치가 완료되었다면, 이제 Oh-My-OpenCode를 사용해봅시다!

## 🎮 두 가지 작업 모드

Oh-My-OpenCode는 두 가지 주요 작업 모드를 제공합니다:

| 모드 | 특징 | 적합한 상황 |
|------|------|-------------|
| **Ultrawork** | 빠르고 자동화된 | 간단한 기능, 빠른 수정 |
| **Prometheus** | 정밀하고 계획적인 | 복잡한 기능, 대규모 변경 |

## ⚡ Ultrawork 모드

프롬프트에 `ultrawork` 또는 `ulw` 키워드를 포함하면 자동으로 활성화됩니다.

### 사용법

```bash
ulw README.md에 설치 가이드 섹션 추가해줘
```

### 동작 방식

```
사용자: ulw 사용자 인증 기능 추가해줘
         │
         ▼
┌─────────────────────────────────────┐
│  1. 자동 탐색                        │
│     - 기존 코드 분석                 │
│     - 관련 파일 식별                 │
├─────────────────────────────────────┤
│  2. 자동 계획                        │
│     - 구현 방법 결정                 │
│     - 필요한 변경 사항 정리          │
├─────────────────────────────────────┤
│  3. 자동 구현                        │
│     - 코드 작성                      │
│     - 테스트 추가                    │
├─────────────────────────────────────┤
│  4. 자동 검증                        │
│     - 빌드 확인                      │
│     - 테스트 실행                    │
└─────────────────────────────────────┘
```

### 예제

::: code-group
```bash [기능 추가]
ulw 다크 모드 토글 버튼 추가해줘
```

```bash [버그 수정]
ulw 로그인 페이지에서 발생하는 타입 에러 수정해줘
```

```bash [리팩토링]
ulw utils.ts 파일을 여러 모듈로 분리해줘
```

```bash [문서화]
ulw API 엔드포인트들의 JSDoc 주석 추가해줘
```
:::

## 🎯 Prometheus 모드

복잡한 작업에는 Prometheus 모드를 사용하세요. 인터뷰를 통해 요구사항을 명확히 하고, 상세한 계획을 수립합니다.

### 사용법

**Step 1**: Tab 키를 눌러 Prometheus 진입

**Step 2**: 인터뷰 질문에 답변

```
Prometheus: 어떤 작업을 도와드릴까요?
You: 결제 시스템을 추가하고 싶어요

Prometheus: 어떤 결제 수단을 지원할 예정인가요?
You: 신용카드와 PayPal이요

Prometheus: 정기 결제(구독)도 필요한가요?
You: 네, 월간 구독 기능이 필요해요

Prometheus: 기존 사용자 모델과 어떻게 연동할까요?
You: users 테이블에 subscription 필드를 추가해주세요
```

**Step 3**: 생성된 계획 검토

**Step 4**: `/start-work` 명령으로 실행

```bash
/start-work
```

### 언제 Prometheus를 사용할까?

::: tip Prometheus 모드 추천 상황
- 🏗️ 새로운 대규모 기능 개발
- 🔄 복잡한 리팩토링
- 🏛️ 아키텍처 변경
- 📦 여러 파일에 걸친 변경
:::

## 🔑 키워드 치트시트

| 키워드 | 효과 | 예시 |
|--------|------|------|
| `ultrawork`, `ulw` | Ultrawork 모드 | `ulw 버그 수정해줘` |
| `search` | 검색 모드 | `search 인증 관련 코드` |
| `analyze` | 분석 모드 | `analyze 이 에러의 원인` |

## 📝 첫 번째 작업 예제

### 예제 1: README 업데이트 (Ultrawork)

```bash
ulw README.md에 다음 내용 추가해줘:
- 프로젝트 소개
- 설치 방법
- 사용 예제
```

**결과**:
- README.md 분석
- 적절한 위치에 섹션 추가
- 마크다운 포맷팅 적용

### 예제 2: API 엔드포인트 추가 (Ultrawork)

```bash
ulw /api/users/:id 엔드포인트 추가해줘
GET으로 사용자 정보 조회하고, 없으면 404 반환
```

**결과**:
- 기존 API 구조 분석
- 라우터 파일에 엔드포인트 추가
- 타입 정의 추가
- 에러 처리 구현

### 예제 3: 인증 시스템 구현 (Prometheus)

```
[Tab 키]

Prometheus: 어떤 작업을 도와드릴까요?
You: JWT 기반 인증 시스템을 구현하고 싶어요

Prometheus: 어떤 인증 방식을 지원할까요?
You: 이메일/비밀번호 로그인과 소셜 로그인(Google)

Prometheus: 토큰 만료 시간은 어떻게 설정할까요?
You: 액세스 토큰 1시간, 리프레시 토큰 7일

Prometheus: 비밀번호 정책이 있나요?
You: 최소 8자, 대소문자 + 숫자 포함

[계획 생성됨 - 검토 후]

/start-work
```

## 💡 팁

::: tip 효과적인 사용법
1. **간단한 작업**은 `ulw`로 빠르게
2. **복잡한 작업**은 Prometheus로 정밀하게
3. **불확실할 때**는 Prometheus로 시작 (인터뷰가 요구사항 명확화에 도움)
:::

:::
