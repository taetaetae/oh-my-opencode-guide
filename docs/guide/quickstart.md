# 빠른 시작

Oh-My-OpenCode의 핵심은 **두 가지 작업 모드**입니다. 상황에 맞게 선택하세요.

## ⚡ Ultrawork 모드

**키워드 하나로 빠르게 작업**을 시작합니다.

### 사용법

프롬프트에 `ultrawork` 또는 `ulw`를 포함하면 됩니다:

```bash
ulw 다크 모드 토글 버튼 추가해줘
```

### 무슨 일이 일어나나요?

```
ulw 다크 모드 토글 추가해줘
         │
         ▼
    ┌─────────┐
    │  탐색   │ ← 기존 코드 분석, 관련 파일 찾기
    └────┬────┘
         ▼
    ┌─────────┐
    │  계획   │ ← 구현 방법 결정
    └────┬────┘
         ▼
    ┌─────────┐
    │  구현   │ ← 코드 작성
    └────┬────┘
         ▼
    ┌─────────┐
    │  검증   │ ← 빌드/테스트 확인
    └─────────┘
```

### 언제 사용하나요?

- ✅ 명확한 작업 (버그 수정, 간단한 기능 추가)
- ✅ 빠른 프로토타입
- ✅ 문서 업데이트

### 예시

```bash
# 기능 추가
ulw 사용자 프로필에 아바타 업로드 기능 추가해줘

# 버그 수정
ulw 로그인 페이지의 타입 에러 수정해줘

# 리팩토링
ulw utils.ts를 여러 모듈로 분리해줘

# 문서화
ulw API 엔드포인트에 JSDoc 주석 추가해줘
```

## 🎯 Prometheus 모드

**인터뷰를 통해 요구사항을 명확히** 하고 체계적으로 진행합니다.

### 사용법

1. **Tab 키**를 눌러 Prometheus 진입
2. 질문에 답변하며 요구사항 구체화
3. 생성된 계획 검토
4. `/start-work`로 실행

### 인터뷰 예시

```
[Tab 키 누르기]

Prometheus: 어떤 작업을 도와드릴까요?
You: 결제 시스템을 추가하고 싶어요

Prometheus: 어떤 결제 수단을 지원할 예정인가요?
You: 신용카드와 PayPal이요

Prometheus: 정기 결제(구독)도 필요한가요?
You: 네, 월간 구독 기능이 필요해요

Prometheus: 기존 사용자 모델과 어떻게 연동할까요?
You: users 테이블에 subscription 필드를 추가해주세요

[상세 계획 생성됨]

/start-work  ← 계획 실행
```

### 언제 사용하나요?

- ✅ 복잡한 기능 개발
- ✅ 요구사항이 불명확할 때
- ✅ 여러 파일에 걸친 대규모 변경
- ✅ 아키텍처 변경

## 🔑 알아두면 좋은 키워드

| 키워드 | 효과 |
|--------|------|
| `ultrawork`, `ulw` | Ultrawork 모드 활성화 |
| `analyze` | 분석 모드 (코드 분석에 집중) |
| `search` | 검색 모드 (코드베이스 탐색) |

---

## 💡 어떤 모드를 선택할까?

```
작업이 명확한가?
    │
    ├─ Yes → ulw 사용
    │
    └─ No → Tab → Prometheus
              │
              └─ 인터뷰로 요구사항 명확화
```

::: tip 팁
**불확실하면 Prometheus로 시작하세요.** 인터뷰 과정에서 요구사항이 명확해지고, 놓친 부분을 발견할 수 있습니다.
:::
