# 에이전트 시스템

Oh-My-OpenCode의 핵심은 **11개의 전문 에이전트**입니다. 각 에이전트는 특정 분야의 전문가 역할을 하며, 필요에 따라 자동으로 협력합니다.

## 🎭 에이전트 = 전문가 팀

```
일반 AI 어시스턴트              Oh-My-OpenCode
┌─────────────────┐           ┌─────────────────────────────┐
│                 │           │  👔 Sisyphus (팀장)          │
│    하나의 AI    │           │  🏛️ Oracle (아키텍트)        │
│    모든 것을    │    →      │  📚 Librarian (연구원)       │
│     처리       │           │  🔍 Explore (탐색가)         │
│                 │           │  📋 Prometheus (기획자)      │
└─────────────────┘           └─────────────────────────────┘
```

## 🌟 자주 사용하는 에이전트

### Sisyphus - 메인 오케스트레이터

> 기본 에이전트. 작업을 분석하고 적절한 에이전트에게 위임합니다.

**자동으로 하는 일:**
- 작업 분석 및 분해
- 적절한 에이전트에게 위임
- 결과 종합

```
사용자: "인증 시스템 추가해줘"
    │
    ▼
Sisyphus: 작업 분석
    │
    ├─→ Librarian: JWT 라이브러리 조사
    ├─→ Explore: 기존 인증 코드 탐색
    └─→ 직접 구현 또는 Sisyphus-Junior에게 위임
```

---

### Oracle - 아키텍처 상담가

> 복잡한 설계 결정이나 어려운 버그에 대해 상담합니다. **읽기 전용**입니다.

**언제 사용되나요?**
- 아키텍처 결정이 필요할 때
- 복잡한 버그를 디버깅할 때
- 설계 리뷰가 필요할 때

**사용 예시:**
```bash
이 시스템의 확장성 문제를 분석해줘
→ Oracle이 코드베이스 분석
→ 병목 지점 식별
→ 개선 방안 제안 (직접 수정은 안 함)
```

::: warning Oracle은 비용이 높습니다
정말 필요한 상황에서만 사용하세요. 간단한 질문에는 다른 에이전트가 더 적합합니다.
:::

---

### Librarian - 문서/라이브러리 연구원

> 외부 라이브러리 문서를 조회하고, GitHub에서 실제 사용 예제를 찾습니다.

**언제 사용되나요?**
- 새로운 라이브러리 사용법을 알고 싶을 때
- 공식 문서를 찾아야 할 때
- 다른 프로젝트의 구현 예제가 필요할 때

**사용 예시:**
```bash
React Query v5의 useMutation 사용법 알려줘
→ Librarian이 공식 문서 조회 (context7 MCP)
→ GitHub에서 실제 사용 예제 검색 (grep_app MCP)
→ 종합된 가이드 제공
```

---

### Explore - 코드베이스 탐색가

> "이 기능 어디에 있어?" 같은 질문에 빠르게 답합니다. **무료**입니다.

**언제 사용되나요?**
- 특정 기능이 어디에 구현되어 있는지 찾을 때
- 코드 패턴을 검색할 때
- 프로젝트 구조를 파악할 때

**사용 예시:**
```bash
인증 미들웨어가 어디에 있어?
→ Explore가 코드베이스 검색
→ 관련 파일 위치와 컨텍스트 제공
```

::: tip Explore는 무료이고 빠릅니다
코드베이스 탐색이 필요하면 부담 없이 사용하세요. 병렬로 여러 개 실행할 수도 있습니다.
:::

---

### Prometheus - 전략 기획자

> Tab 키로 진입. 인터뷰를 통해 요구사항을 명확히 하고 상세한 계획을 수립합니다.

**언제 사용되나요?**
- 복잡한 기능 개발
- 요구사항이 불명확할 때
- 대규모 변경 작업

**사용 방법:**
```
[Tab 키]
Prometheus: 어떤 작업을 도와드릴까요?
You: 결제 시스템을 추가하고 싶어요
... (인터뷰 진행)
[계획 생성]
/start-work
```

## 📊 에이전트 비용 가이드

| 에이전트 | 비용 | 용도 |
|----------|------|------|
| **Explore** | 무료 | 코드 탐색, 파일 찾기 |
| **Librarian** | 💰 | 문서 조회, GitHub 검색 |
| **Multimodal-Looker** | 💰 | 이미지/PDF 분석 |
| **Sisyphus-Junior** | 💰💰 | 작업 실행 |
| **Sisyphus** | 💰💰💰 | 메인 오케스트레이터 |
| **Oracle** | 💰💰💰 | 아키텍처 상담 |
| **Prometheus** | 💰💰💰 | 전략 기획 |

---

## 🔒 에이전트별 제한

각 에이전트는 역할에 맞는 권한만 가집니다:

| 에이전트 | 할 수 있는 것 | 할 수 없는 것 |
|----------|---------------|---------------|
| **Oracle** | 읽기, 분석, 조언 | 코드 수정, 파일 생성 |
| **Librarian** | 문서 조회, 검색 | 코드 수정 |
| **Explore** | 코드 탐색, 검색 | 코드 수정 |
| **Sisyphus-Junior** | 코드 작성, 수정 | 다른 에이전트에게 위임 |

## 💡 효과적인 사용 팁

::: tip 이렇게 사용하세요
- **코드 찾기** → Explore (무료, 빠름)
- **라이브러리 질문** → Librarian (문서 + 예제)
- **설계 고민** → Oracle (비용 높지만 정확)
- **복잡한 작업** → Prometheus (인터뷰 → 계획)
:::

::: danger 피해야 할 패턴
- ❌ 간단한 질문에 Oracle 사용 (비용 낭비)
- ❌ 에이전트 결과를 검증 없이 신뢰
- ❌ 병렬 처리 가능한 작업을 순차적으로 요청
:::
